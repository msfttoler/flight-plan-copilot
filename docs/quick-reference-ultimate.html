Pro Tips
  • Open files first — Copilot reads your open tabs
  • Name well — getFlightPrice() > gfp()
  • Comment first, let Copilot write the code
  • Highlight code before ⌘I for targeted edit
  • Combine @workspace + #file for precision
  • "Don't change X" constrains Copilot's scope
  • Iterate — rough prompt → refine in follow-ups
  • "Explain this error" + paste = fast debug
  • Edits mode for cross-file refactors
  • Review always — you are the editor, not the typist
Pitfalls
  • Vague prompt → vague code. Be specific!
  • Copilot can't read files you haven't opened
  • Always check logic, edge cases, security
  • Break big asks into smaller conversational steps
