## The Five Things to Remember (3 min)

> "If you forget everything else from today, remember these five things:"

**1. Comments are prompts.**
Write what you want before you write how to do it. The better your comment, the better the code.

**2. Context is king.**
Open relevant files before asking. Use `@workspace` for project-wide questions. The more Copilot knows, the better it performs.

**3. Iterate, don't one-shot.**
Don't try to get the perfect result in one prompt. Start rough, refine in conversation. That's how the best developers use it.

**4. Always review.**
Copilot is a draft machine. You are the editor. Read what it generates. Run the tests. Trust but verify.

**5. It gets better the more you use it.**
Copilot learns your patterns within a session. Your prompting skills improve with practice. Day 30 of Copilot is dramatically better than day 1.

## Q&A (5 min)

Open the floor. Common questions and suggested answers:

**"Is my code being used to train the model?"**
> "GitHub Copilot Business and Enterprise do not use your code for training. For individual plans, you can opt out in settings. Check your organization's policy."

**"Can I use Copilot-generated code in production?"**
> "Yes. You own the code. But treat it like any code — review it, test it, make sure it meets your standards. The same way you'd review a PR from a colleague."

**"What about security / secrets?"**
> "Copilot has a secret detection filter. But still: never hardcode secrets, always use environment variables, and let your security scanning tools do their job."

**"Does it work with [my language/framework]?"**
> "Copilot works with virtually every language. It's best with popular languages (JS/TS, Python, Java, C#, Go, Ruby, Rust). It's less good with niche or very new languages. Try it and see."

**"How is this different from ChatGPT?"**
> "Integration. Copilot lives in your editor, sees your code, understands your project. ChatGPT is a general assistant you copy-paste to. For coding, the integrated experience is dramatically better."
