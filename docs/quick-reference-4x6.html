Tips, Patterns & Decision Tree

Back Flight Plan Dev Day
Pro Tips
  • Open relevant files before chatting — Copilot uses open tabs as context
  • Name things well — getFlightPrice() prompts better than gfp()
  • Comment first, then pause — let Copilot write the implementation
  • Highlight code before ⌘I for targeted transforms
  • Use #file to point chat at specific files for context
  • Iterate — start with a rough prompt, refine in follow-ups
  • Say "don't change X" to constrain what Copilot touches
  • Ask "explain this error" + paste the error message — fast debugging
  • Use edits mode for changes across multiple files at once
  • Review always — Copilot is a first draft, you are the editor
Common Pitfalls
  • Vague prompts → vague code. Be specific about inputs and outputs
  • Forgetting context — Copilot can't read files you haven't opened
  • Accepting without reading — always check logic and edge cases
  • Giant prompts — break big tasks into smaller conversational steps
Prompt Patterns That Work
Comment → Code
// Validate boarding pass: check flight, date, passenger name, seat
Test → Implementation
"Here's my test file. Write the function that makes these tests pass."
Old → New
"Refactor this callback-based code to use async/await"
Error → Fix
"I'm getting [paste error]. Here's the relevant code: [#selection]"
Explain → Learn
"Explain this regex step by step and suggest improvements"
Scaffold → Build
"Create a REST endpoint for flight search with query params: origin, destination, date"
